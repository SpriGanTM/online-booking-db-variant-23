## Вопрос

Опишите разницу между представлением (VIEW) и материализованным представлением. Могли бы вы использовать VIEW для упрощения вашего первого запроса (с тремя JOIN)?

## Ответ

VIEW представляет собой виртуальную таблицу, которая хранит не данные, а SQL-запрос. При обращении к VIEW MySQL каждый раз выполняет запрос заново и получает актуальные данные из связанных таблиц.

Материализованное представление хранит уже готовый результат запроса в отдельной структуре. Это позволяет ускорить чтение данных, однако требует периодического обновления.

В MySQL полноценные материализованные представления отсутствуют, поэтому чаще используют обычные VIEW либо создают отдельные таблицы с автоматическим обновлением.

В моей базе данных можно использовать VIEW для упрощения JOIN-запроса между таблицами repairs, bicycles, clients, services и masters. Это позволит обращаться к готовому представлению вместо написания длинного SQL-запроса.

Пример:

CREATE VIEW repair_info AS
SELECT ...
FROM repairs
JOIN bicycles
JOIN clients
JOIN services
JOIN masters;

После этого можно выполнять:

SELECT * FROM repair_info;
